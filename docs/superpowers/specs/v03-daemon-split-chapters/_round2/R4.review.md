### Gate (a) lint:no-ipc — **NEEDS-FIX (P1)**

Round-1 ask: canonical script + ESLint backstop + `.no-ipc-allowlist`, all three named, allowlist contents documented.

- Canonical script `tools/lint-no-ipc.sh`: PRESENT (chapter 12 §4.1, cross-referenced from chapter 08 §5h, chapter 11 §6).

- `tools/.no-ipc-allowlist` referenced: PRESENT.

**P1-R2-12-a.1 (allowlist mechanism is broken-by-design)**: chapter 12 §4.1 documents the allowlist format as `path:line-number-range` (e.g. `packages/electron/src/preload/preload-descriptor.ts:1-40`) but the bash implementation is `grep -vFf <(awk -F: '{print $1}' tools/.no-ipc-allowlist)`. `awk -F: '{print $1}'` strips the line-range, leaving just the path; `grep -vFf` then suppresses ALL hits in that file (every line of `preload-descriptor.ts`), not just lines 1–40. A typo or accidental `ipcMain` use anywhere in an allowlisted file passes silently. Either:
- Implement the line-range filter in the script (subtract by `path:line` after numbering hits — non-trivial bash; consider a small node script), OR
- Change documented format to "whole files only" and drop the `:1-40` example, AND add a sibling assertion that allowlisted files are `< N` lines so accidental growth is reviewed.

**P1-R2-12-a.2 (allowlist contents are not enumerated, only the format)**: the round-1 ask was "allowlist contents documented." Chapter 12 §4.1 documents the format and the ESLint message says "sanctioned exceptions go through tools/.no-ipc-allowlist (descriptor preload only)" — but the actual list of paths permitted in v0.3 is not pinned anywhere. F2/F3 selected the descriptor-preload path; the v0.3 allowlist is a single-line file. Pin it: "v0.3 `tools/.no-ipc-allowlist` contents are exactly `packages/electron/src/preload/preload-descriptor.ts` (one line). Any addition is a chapter-15 forever-stable touch and requires R4 sign-off." Without this, "0 IPC residue" is unprovable because the gate's allowlist is unbounded.

**P1-R2-12-a.3 (ESLint config file extension drift)**: chapter 12 §4.1 says `packages/electron/.eslintrc.cjs` (legacy ESLint v8). Chapter 11 §5 says `packages/electron/eslint.config.js` (flat config v9). Pick one in chapter 12 §4.1 to match chapter 11 §5 OR vice-versa; otherwise CI step "ESLint backstop in .eslintrc.cjs" never fires under flat-config ESLint, and the gate passes vacuously.

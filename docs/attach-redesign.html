Attach 重新设计 —— UX 模型

右侧内容区怎么呈现 session。只解两个问题:第一次别 flicker,看过的切回要瞬间。
范围:pty attach + xterm mount · 历史时间线 docs/attach-timeline-review.html

1 · UX —— 三种状态

2 · 两条规则

资源策略,一句话。

规则 1
没看过 = 零资源

没点过的 session 完全不准备 —— 不订 PTY,不建 xterm,什么都不占。

规则 2
看过 = 留着不放

点过一次之后 xterm 和 PTY 订阅都留住。关 ccsm 才一次性回收。

3 · 怎么做

三块东西凑起来,不展开。

  • z-stack —— 右侧 host 里挂 N 个 wrapper,每个 wrapper 内有自己的 xterm。切换只改 display 和 z-index,xterm 实例从不在 wrapper 之间 reparent。这是"看过的切回瞬间、位置不变"的根本。
  • per-shell mask —— 每个 wrapper 自带一个遮罩 div。冷启动期间这个 wrapper 的遮罩亮,其它 wrapper 的遮罩状态各自独立,切走切回都不串。
  • 服务端 snapshot —— main 进程用 @xterm/headless + SerializeAddon 保留每个 session 的镜像,renderer 冷启动时拉一次写进 xterm 就行。

4 · 其他操作

delete / reload / copy,各一句话,不引入新概念。

delete
移除

移除 shell + wrapper。删的是顶层 → z-stack 自动塌到下一层;塌到空 → 回 State 0 黑屏。

reload
原地刷

term.reset() + 重拉 snapshot + 写入,不 dispose。顶层显示遮罩,非顶层静默走完不显示。

copy
新 session

调用方拿到新 sid。用户点了才 attach,走正常冷启动。

5 · 兜底

异常处理的原则。

任何状态卡住或异常,dispose 这个 shell + 重新 cold start。不在 design 里穷举各种 race / cancellation / ordering 细节,实现者自己想 —— 反正最坏路径就是回到第一次点击那条路径。

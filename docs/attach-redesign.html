Attach 重新设计 —— UX 模型

右侧内容区怎么呈现 session,以及背后大致是怎么做到的。3 分钟看完。
范围:pty attach + xterm mount + 侧边栏耦合 · 配套文档 docs/attach-timeline-review.html(历史)

1 · UX 模型 —— 三种状态

右侧内容区就这三种样子。左边的 sidebar 是固定的。

State 0

刚打开 ccsm

session A
session B
session C
— 空 —
右侧全黑。还没人 attach 过,什么都没准备。
State 1

冷启动中

session A
session B
session C
(底层旧 shell,被盖住)
准备中…
遮罩盖住右侧。后台拉 snapshot、建 xterm、写入、滚到底,好了揭开。
State 2

已 attach 切换

session A
session B
session C
session C
session B
session A · 顶层
$ npm test
PASS ./foo.test.ts
▍
右侧是 z-stack。点谁谁拉到顶,各自保留上次离开的位置,无遮罩、无冷启动。

状态怎么流转

State 0  ──点第一个 session──▶  State 1  ──准备完──▶  State 2
                                                       │
                                                       ├──点已看过──▶ 留在 State 2(秒切)
                                                       │
                                                       └──点没看过──▶ State 1(再来一次冷启动)

2 · 两条规则

用户行为只分两种,响应也只有两种。

规则 1
第一次点 = 冷启动

右侧盖一层 "准备中" 的遮罩。背后拉 snapshot、建 xterm、写入、滚到底,全部做完才揭开。用户不会看到半成品。

规则 2
再点已看过的 = 秒切

右侧 z-stack 里把这个 session 翻到顶,viewport 还停在上次离开的位置。无遮罩,无加载,无重建。

3 · 背后做什么(high-level)

不展开 IPC 和状态机,只说大方向。

  • 没点过的 session 完全不准备 —— 不订 PTY,不建 xterm,什么资源都不占。
  • 点过一次之后 留住不放 —— xterm 实例和 PTY 订阅都不 dispose,所以再切回来是秒级。
  • 关 ccsm 才回收 —— 退出时一次性 dispose 所有留下来的 shell。
  • 右侧是 z-stack —— 看过的 shell 都叠在那里,点哪个哪个拉到顶。
  • 看过就留,关 ccsm 一次性回收 —— 不做缓存上限,不做淘汰。

5 · 其他操作的行为

sidebar 上的三个操作:删除、重做、复制。都不引入新的 UX 概念,只是套用前面两条规则。

操作 1
删除 (delete)

· sidebar 那行消失。
· 后台 dispose 这个 session 的 xterm,解订 PTY。
· 删的是当前顶层 → 顶层移走,z-stack 下一层自动露出(上次看的那个)。
· 删的是 z-stack 里非顶层 → 用户看不到任何变化。
· 删的是从没看过的 session → 啥都不发生,本来就没建。
· 删的是正在冷启动的 session → cancel,遮罩消失,顶层退回下一层。

操作 2
重做 (reload)

· sid 不变,xterm 实例复用,不重建,不破坏 z-stack 缓存。
· 走和第一次点一样的冷启动流程:盖遮罩 → 清屏 + 重拉 snapshot + 写入 + 滚到底 → 揭开。
· 视觉上等于这个 session 当场重做一次冷启动。
· z-stack 里其他 shell 不受影响,顺序不变。

操作 3
复制 (copy session)

· 调用方拿到新 sid,这部分是已有产品行为,不改。
· 新 sid 通常会立刻被打开 → 走第一次点击的冷启动路径。
· 原 session 在 z-stack 里照常保留,顺序不变。
· 设计上不需要特殊路径,等同于自动 click(newSid)。

6 · 实现要点

给后续 worker 看的落地清单。不是完整方案,只是关键决策。

替换掉的现有代码

  • src/terminal/usePtyAttach.warm.ts —— 19 步过程整体替换为 "点击 → 翻 z-stack 顶层"。
  • xtermWarmRegistry.ts 里的 cold-vs-warm 分支 —— 不再区分,统一走 "shell 不存在就建,存在就拉顶"。
  • buffering listener 模式 —— 没用了。点击之前根本不订 PTY,点击之后订一次就一直挂着。

z-stack 怎么实现

右侧 host 里放 N 个 wrapper div,每个 wrapper 里挂一个 xterm。切换只动 display 和 z-index:目标 shell display:'',其余 display:'none'。xterm 实例从不在 wrapper 之间 reparent。

遮罩怎么做

在右侧 host 上盖一个 position:absolute; inset:0 的遮罩 div。冷启动开始时 display:'' 显示 "准备中…",背后建 shell + 写 snapshot + 滚到底全部跑完后 display:'none' 揭开。秒切场景遮罩自始至终 display:'none'。

PTY 订阅时机

第一次点 session 时调 pty.open(sid) 订上,之后这个订阅一直挂着,直到关 ccsm 调 pty.close。脱离前台不取消订阅,chunk 持续写入对应的 xterm scrollback —— 这样秒切回来时新内容已经在那里了。
